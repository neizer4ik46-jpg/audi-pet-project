# Руководство по разработке (Contributing Guide)

## Как начать разработку

### Предварительные требования
- Git установлен на вашем компьютере
- Текстовый редактор или IDE (VS Code рекомендуется)
- Браузер с поддержкой HTML5 и CSS3

### Клонирование репозитория
```bash
git clone https://github.com/neizer4ik46-jpg/audi-pet-project.git
cd audi-pet-project
```

## Структура веток (Branch Strategy)

- **master** - основная ветка, содержит стабильный рабочий код
- **develop** - ветка разработки для новых функций
- **feature/*** - ветки для отдельных функций (например: `feature/responsive-design`)
- **bugfix/*** - ветки для исправления ошибок (например: `bugfix/header-alignment`)

## Процесс разработки

### 1. Создание новой ветки
```bash
git checkout -b feature/your-feature-name
```

### 2. Внесение изменений
- Вносите логичные, сознательные изменения
- Тестируйте в разных браузерах (Chrome, Firefox, Safari, Edge)
- Убедитесь, что код валиден и читаем

### 3. Коммиты
```bash
git add .
git commit -m "feat: краткое описание изменений"
```

**Формат сообщений коммитов:**
- `feat:` - новая функция
- `fix:` - исправление ошибки
- `docs:` - изменения документации
- `style:` - изменения стилей (CSS)
- `refactor:` - рефакторинг кода
- `perf:` - улучшения производительности
- `test:` - добавление или изменение тестов

### 4. Push и Pull Request
```bash
git push origin feature/your-feature-name
```

Затем создайте Pull Request в GitHub с описанием изменений.

## Стандарты кодирования

### HTML
- Используйте семантическую разметку HTML5
- Проверьте валидность с помощью W3C Validator
- Используйте правильные ARIA атрибуты для доступности

### CSS
- Следуйте структуре файла reset.css и style.css
- Используйте классы для стилизации (избегайте ID селекторов)
- Комментируйте сложные CSS правила
- Обеспечите кроссбраузерную совместимость

### Изображения и медиа
- Оптимизируйте изображения перед добавлением
- Используйте форматы: PNG для графики, JPG для фотографий, MP4 для видео
- Помещайте копии в `images/copy/`, оригиналы в `images/original/`

## Тестирование

Перед тем как создавать Pull Request:
1. Протестируйте в разных браузерах
2. Проверьте адаптивность на мобильных устройствах
3. Убедитесь, что нет консольных ошибок
4. Проверьте производительность

## Вопросы и поддержка

Если у вас есть вопросы:
1. Посмотрите в Issues на GitHub
2. Создайте новый Issue с подробным описанием
3. Обозначьте версию браузера и ОС

## Лицензирование

Внося изменения в этот проект, вы соглашаетесь с условиями разработки учебного проекта.
